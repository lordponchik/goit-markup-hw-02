## Eligibility criteria for a mentor

### Project

**`¬´A1¬ª`** At the root of the project there is a folder `images` with images.

**`¬´A2¬ª`** At the root of the project there is a `css` folder with style files.

**`¬´A3¬ª`** All styles are written in one `styles.css` file, which is located in the `css` folder.

**`¬´A4¬ª`** There are no capital letters, spaces and transliteration in the file names, only letters and words of the English language.

**`¬´A5¬ª`** Source code formatted with `Prettier`.

**`¬´A6¬ª`** All images and text content are taken from the layout.

**`¬´A7¬ª`** All bitmaps are optimized using
[squoosh](https://squoosh.app/).

**`¬´A8¬ª`** The code is written as follows [**manual**](https://codeguide.co/).

### Markup

**`¬´B1¬ª`** The `Portfolio` page markup is typed in the file `portfolio.html`.

**`¬´B2¬ª`** Completed HTML markup of all layout elements.

**`¬´B3¬ª`** Tags are used according to their semantic meaning.

**`¬´B4¬ª`** HTML passes validation [**validator**](http://validator.w3.org/nu/) no mistakes.

**`¬´B5¬ª`** Class names are descriptive and public to another developer.

**`¬´B6¬ª`** Class names do not contain capital letters, spaces, transliteration, and tag names, only English letters and words. If the class name is of several words, they are separated by a hyphen.

**`¬´B7¬ª`** The `href` attribute of the `Studio` and `Portfolio` navigation links contains relative path to the HTML files of these pages. When you click on the link, go to the corresponding page in the current browser tab.

**`¬´B8¬ª`** The `<img>` tags have size attributes, at least `width`.

**`¬´B9¬ª`** Images are exported from the layout in the format `jpg`.

**`¬´B10¬ª`** Groups of elements of the same type are collected in lists `<ul>`.

**`¬´B11¬ª`** The filter on the `Portfolio` page is made by a list of buttons, each of which have the `type="button"` attribute set.

**`¬´B12¬ª`** The header and footer layout is the same on all pages.

**`¬´B13¬ª`** All the fonts required for the layout and their variations (weight and style) connected from the Google Fonts service with one link. Required weight for `Raleway` ‚Äì 700, and for `Roboto` - 400, 500, 700 and 900.

**`¬´B14¬ª`** Inside the button markup, there are no additional elements, such as spans or links.

## Formalization

**`¬´C1¬ª`** No global element styles other than `<body>`.

**`¬´C2¬ª`** Class selectors are used for styling.

**`¬´C3¬ª`** Styles are missing `!important`.

**`¬´C4¬ª`** For interactive elements (buttons and links), when hovering with the mouse or focus from the keyboard, there is an active state specified in the layout (change colors).

**`¬´–°5¬ª`** Contact text in header and footer changes color on hover and focus.

**`¬´C6¬ª`** To store layout color palette (text, background, selection) CSS variables are used.

**`¬´–°7¬ª`** The `<body>` element has its `font-family` property set to dominant layout in `Roboto` font.

**`¬´–°8¬ª`** Alternate font options and family type specified (sans-serif) at the end of the `font-family` enum of the `<body>` element.

**`¬´–°9¬ª`** The `Roboto` font-family is explicitly set only for the `<body>` element, other elements inherit it.

**`¬´–°10¬ª`** The `<body>` element has its `color` property set to dominant layout with text color. The rest of the text inherits or overrides this value.

**`¬´–°11¬ª`** Font size (`font-size` property) of all text elements exactly matches the values from the layout.

**`¬´–°12¬ª`** The line height (`line-height` property) of all text elements exactly matches the values from the layout and is specified as a multiplier, not in `px`.

**`¬´–°13¬ª`** The color (`color` property) of all text elements exactly matches values from the layout.

**`¬´–°14¬ª`** Font weight (`font-weight` property) of all text elements exactly matches the values from the layout.

**`¬´–°15¬ª`** The weight of the font (the `font-weight` property) is only explicitly specified if the value in the layout differs from the standard for this element in the browser.

**`¬´–°16¬ª`** The buttons have a `cursor` property set to `pointer`.

**`¬´–°17¬ª`** Styles do not repeat property values that are set by the browser by default. For example, links do not need to be specified with `cursor: pointer`, but paragraphs `font-style: normal` or `font-weight: 400`.
